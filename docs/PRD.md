# 2. PRD (Product Requirements Document)
**제품명:** CampX MVP
**목표:** 단일 캠핑장 대상 PoC 성공 및 AI 추천 정확도 검증

## 2.1. 제품 범위 (MVP Scope)
복잡한 기능은 모두 덜어내고, **"입실부터 퇴실까지의 가이드"**에만 집중합니다.

| 우선순위 | 기능명 | 상세 요구사항 |
| :--- | :--- | :--- |
| **P0 (Critical)** | **AI 맞춤형 스케줄러** | - 사용자 입력(동반자, 피로도, 선호) 벡터화<br>- `pgvector` 기반 프로그램 매칭<br>- 타임라인(시계열) 형태의 일정표 생성 및 출력 |
| **P0 (Critical)** | **2.5D 인터랙티브 맵** | - 특정 캠핑장 항공뷰 기반 맵핑<br>- 주요 시설(화장실, 개수대, 내 텐트) 핀(Pin) 표시<br>- 클릭 시 상세 정보 모달 (HTMX 활용) |
| **P1 (High)** | **비대면 체크인** | - 예약 정보 기반 QR 코드 발급<br>- 관리자용 QR 스캔 페이지 (별도 앱 개발 X, 모바일 웹) |
| **P1 (High)** | **운영 알림 봇** | - 매너타임 시작, 예약된 체험 시간 임박 시 알림 (카카오 알림톡 연동) |
| **P2 (Medium)** | **피로도 기반 Upselling** | - AI가 계산한 '지침 지수'가 높을 때, '장작 배달' 버튼 노출 |

## 2.2. 사용자 시나리오 (User Journey)
1.  **Onboarding:** 예약 확정 문자 링크 클릭 → 별도 앱 설치 없이 웹으로 접속.
2.  **Profiling:** "오늘 컨디션은 어떠신가요?" 등 간단한 3가지 질문 답변.
3.  **Simulation:** AI가 제안한 1박 2일 일정 확인 및 수정.
4.  **Arrival:** 캠핑장 도착 → 관리동 방문 없이 모바일 체크인.
5.  **Stay:** 2.5D 지도를 보며 화장실 이동, AI 알림에 맞춰 체험존 방문.
6.  **Exit:** 체크아웃 버튼 클릭 및 경험 평가(데이터 재학습용).

## 2.3. 성능 및 품질 요구사항
*   **AI 응답 속도:** 스케줄 생성 로직 3초 이내 완료.
*   **모바일 호환성:** 별도 앱(iOS/Android) 없이 모바일 브라우저(Safari/Chrome)에서 네이티브 앱 수준의 UX 제공 (PWA 고려).
